
# v0.4.0 plan — agent intelligence + session power tools

primary theme: **make agents smarter and sessions more flexible**. this doc is now the active remainder for v0.4.0: shipped work is marked done, and the open sections below describe only what still needs to be built or tightened.

---

## shipped in this branch

### done: conflict detection MVP

parallel agent merges now have a safety layer before merge:

- cross-agent diff analysis runs before reviewed agent output is merged
- overlapping file/hunk edits are detected
- non-blocking cases can auto-merge
- blocking overlaps are surfaced in the TUI and held for user decision

### done: cross-model critique phase (roundhouse upgrade)

roundhouse now has an explicit critique pass before synthesis:

- planner outputs are collected first
- critique rounds can run before final synthesis
- critique output is visible in the TUI/sidebar flow
- `/roundhouse --no-critique` is supported

### done: custom agents core

custom agent definitions now exist and are usable:

- agents load from `.caboose/agents/` and global config agents dirs
- markdown frontmatter format is supported
- slash resolution and autocomplete include custom agents
- `spawn_agent` can target a named custom agent
- per-agent model preference, tool allow/deny lists, and `worktree: false` are implemented
- agent directories are auto-created on startup

### done: session fork

forking is now part of the session model:

- `/fork` creates a new session from the current one
- full message history is copied into the fork
- fork metadata is stored in session persistence
- fork context includes a structured handoff summary
- session UI shows fork relationships

### done: session pins

session-level pins now exist:

- `/pin`, `/pins`, and `/unpin` are implemented
- pins are injected into the active system prompt
- pins are visible in the session UI

note: current behavior persists pins in session storage. if we still want truly ephemeral pins, that is follow-up work rather than missing implementation.

### done: conflict detection hardening (2026-03-16)

semantic conflict detection and merge sweep hardening:

- three-tier resolution model: AutoMerge / AutoReconcile / RequiresReview (replaces binary Warn/Block)
- function-level symbol extraction across Rust, Python, Go, JS/TS with `detect_changed_symbols`
- signature change detection (`detect_signature_changes`) catches parameter/return-type drift in diffs
- cross-file interface drift detection (`detect_interface_conflicts`) — if agent A changes a function signature and agent B calls that function, it's flagged RequiresReview
- shared-symbol conflicts (same symbol touched by multiple agents) → AutoReconcile when hunks don't overlap
- conflict report now shows diff snippets and symbol-level detail
- merge sweep re-collects diffs at merge time via `collect_review_agent_changes` instead of relying on stale data from spawn time
- agents with missing diffs are put into Conflict state instead of silently skipped
- post-merge safety net catches agents that remain in Review after the sweep

### done: worktree tool path normalization (2026-03-16)

subagents in worktrees now resolve tool paths correctly:

- `normalize_tool_arguments` rewrites relative paths in tool calls against the agent's `primary_root`
- covers read_file, write_file, edit_file, glob, grep, diagnostics, run_command, apply_patch
- `apply_patch` receives an explicit `root` parameter and resolves all file entries against it
- `run_command` defaults `cwd` to the worktree root
- subagent worktree changes are committed before diff collection (`commit_worktree`)

### done: paste detection (2026-03-16)

paste-like input no longer accidentally submits:

- rapid character input (>24 chars within 180ms gaps) activates paste-like mode
- Enter key during paste-like mode inserts a newline instead of submitting
- 900ms grace window after bursts prevents chunked Enter events from triggering submit
- covers both home screen and active chat input paths

### done: tracing cleanup (2026-03-16)

- TUI mode now routes logs to sink instead of stderr (was leaking raw log lines under crossterm alternate screen)
- debug mode and non-interactive mode still log to stderr

### done: subagent noop retry (2026-03-16)

- subagents that produce no tracked file changes on tasks that likely require edits get one automatic retry with stricter instructions
- `task_likely_requires_changes` heuristic checks for edit tool names in the task string
- `produced_changes` field added to `SpawnAgentResult` for explicit tracking
- tasks that genuinely produce no changes (e.g. research) are marked Done, not Failed

### done: TUI polish

- input footer split into two rows to prevent keybind hints from clipping on narrow terminals
- keybind hints right-aligned in the info bar
- home screen input height fixed to match two-row info bar
- pin bar is collapsible between header and chat area

### done: roundhouse TUI rendering

- tool calls show running/completed states with colored bullets during planning
- model names displayed in sidebar with purple accent
- streaming text renders across tool rounds instead of waiting for completion

### done: codebase maintainability (2026-03-16)

- tool name constants centralized into `tools/names.rs` — single source of truth, no more scattered string literals
- shared slash command dispatch extracted into `handle_shared_slash` — eliminates duplicated match arms across home/chat
- 16 dead functions removed along with their tests
- workspace boundary enforcement in `apply_patch` — paths validated against workspace root
- dead `recall` tool code and stale `--persist` circuit flag removed
- conflict test scratch files cleaned up

## remaining work

### done: conflict detection follow-up (2026-03-16)

- merge policy by mode: create mode now holds AutoReconcile overlaps for user review; chug mode auto-merges them
- regex caching in `capture_decl` via thread-local `HashMap<&'static str, Regex>` — compiles each pattern once
- `parse_hunk_header` off-by-one fixed: `start` clamped to `max(1)` alongside existing `end` clamp

### done: custom agents UX polish (2026-03-16)

- `/agents` command opens a dialog overlay listing all loaded agents with name, description, model, source
- `auto_approve` field added to agent frontmatter — agents with `auto_approve: true` skip tool approval
- `/agent <name> <task>` already works via direct slash invocation — no separate alias needed
- TOML support deferred — markdown frontmatter covers the use case

### done: agent handoff polish (2026-03-16)

- model switch summary injection: auto-generates handoff context when switching models mid-session
- `Ctrl+H` opens model picker, spawns subagent with selected model + handoff summary as context
- handoff summary includes: user messages, files modified with +/- stats, tool counts, open tasks, focus

### done: agent migration in `/migrate` (2026-03-16)

- Claude Code: scans `.claude/agents/` (global + project), parses markdown agent files
- Open Code: scans agent definitions from opencode config + project `.opencode/agents/`
- maps imported agent definitions into `.caboose/agents/` format via `render_caboose_agent_markdown`
- preview with same checklist/toggle flow used by existing migration UX
- agent definitions auto-reload after migration applies
